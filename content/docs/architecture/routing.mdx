## What is a route?

A route is a URL path that maps to a page or API handler.

- Page route: returns HTML/UI
- API route: returns JSON/status codes

## Main page routes

- `/`: home page
- `/course-rating`: course list page
- `/course-rating/search?query=`: search result page
- `/courses/[courseCode]`: dynamic course detail page (`[courseCode]` is a variable)
- `/submit-comment`: authenticated comment form
- `/about`, `/activities/[slug]`
- `/forum` (currently placeholder)
- `/profile/[[...profile]]`, `/sign-in/[[...sign-in]]`, `/sign-up/[[...sign-up]]`

## API routes

- `GET /api/courses`
- `GET /api/professors`
- `POST /api/comments`

## Compatibility route

`/CourseDetail/[courseCode]` re-exports the canonical detail page.

Why: old/legacy links still work while new canonical route remains `/courses/[courseCode]`.
